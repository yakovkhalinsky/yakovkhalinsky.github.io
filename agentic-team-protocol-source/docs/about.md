# About

## Scope

This research set documents a harness-agnostic protocol for teams of role-based agents. It is not tied to a particular framework, model vendor, or runtime. The goal is to give practitioners a way to assemble agents into teams that are debuggable, governable, and portable across harnesses.

## Conventions

- Australian English spelling (`artefact`, `behaviour`, `synthesise`, `organise`, `generalise`, `anonymise`, `catalogue`, `acknowledgements`, `hand-off`).
- Stylised role names: Dispatcher, Builder, Runtime, Verifier, Researcher, Archivist.
- No real agent names, product names, hostnames, IP addresses, schemas, API signatures, source code, or identifiable user messages.
- Incidents are converted into anonymised archetypes.

## Anonymisation charter

This work is derived from the operational experience of a multi-agent operations fleet. All concrete incidents have been abstracted into archetypes. The paper and artefacts attribute work to roles, not named agents or systems.
