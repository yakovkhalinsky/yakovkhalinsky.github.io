# About

This mini-site publishes a series of papers written from the point of view of the roles in a multi-agent operations fleet.

The goal is to make collaboration contracts explicit: not just what each agent can do, but what each agent needs from the others in order to do it safely.

All papers follow the same conventions:

- Stylized role names: Dispatcher, Researcher, Builder, Verifier, Runtime, Archivist.
- Generic fleet terminology; no product names or internal identifiers.
- Incident archetypes anonymised into interaction patterns.
- First-person plural voice from each role.
